Mimosa

A modern browser stack toolkit. Watch. Compile. Lint. Optimize. Serve. Loaded with JavaScript, CSS, and micro-templating compilers, and goodies like live reload, and first-class RequireJS support.

Jammed with Goodness

Coding in Seconds

Use Mimosa's command line to build out a skeleton app that is ready to go with your desired meta-languages, a bit of sample code, and an server too! No additional installs or plugins needed to get everything Mimosa has to offer.

Watch Assets

Everything happens when you save your code. Mimosa will keep track of your assets and when you save something, kick off all sorts of awesome.

Immediate Feedback

Code doesn't compile? You'll find out right away. When code breaks Mimosa will write big angry messages to the console and use Growl if you've got it installed.

Asset Serving

Mimosa will serve your assets for you if you'd like, or it will invoke your Express server (or the Express server it gave you) so that you only need a single process running to watch and serve.

Configurable, but Not

Mimosa is heavily configurable, but don't worry about it. The configuration starts empty. Sane defaults keep you coding instead of figuring out config. You may never look at or touch the stuff!

Package Management

Mimosa wraps basic Volo add functionality to allow you to easily pull in and install known GitHub dependencies right to the directory you want them.

Live Reload

Both the server built into Mimosa and the one it provides you on project creation come with Live Reload ready to go. When an asset is compiled, your browser refreshes immediately.

Optimize + minify, later

Have the best of both worlds. Develop with unminified, uncompressed, individual assets. Flip a switch and develop with minified and compressed assets that have been merged with RequireJS.

Gzipped, Cache-Busted

All your assets will be gzipped, keeping load times as fast as possible, and cache-busted, saving you from dealing with nagging browser cache issues.

Lint it, Hint it

All of your compiled JS and CSS will be run through JSHint and CSSLint which will encourage you to keep your code as idiomatic as possible.

We got compilers

JavaScript

Mimosa will compile your CoffeeScript and Iced CoffeeScript.

CSS

Stylus w/nib. SASS w/Compass. LESS. Mimosa won't bother compiling your partials individually, but when your partials change it'll compile whichever files at the root of your CSS tree need compiling.

Micro-Templates

Dust. Handlebars. Hogan. Underscore. LoDash. Jade. Just HTML. Mimosa will pre-compile all of your templates.

Meta-Languages? Pfft.

Plain JS, Plain CSS, no templates? You're a classic and Mimosa's cool with that. Files that don't need compiling will just be copied, but they'll still get the royal treatment, including linting and RequireJS verification.

Like RequireJS?

Dependency Checking

Mimosa will keep track of your dependency graph and alert you right away if you've mis-typed a path, referred to something that doesn't exist, or have a circular dependency.

Shim, Map, Plugins, Paths

Yep, yep, yep, yep. It'll check those too. Paths themselves, shims, shim dependencies, map modules and map paths, and plugin paths.

Optimization

If you use RequireJS, you know about the optimizer. With the tick of a flag Mimosa will run the optimizer on every successful JavaScript compile or copy. And you won't need to configure a thing.
