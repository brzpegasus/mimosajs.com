Installation it's a piece of cake

1) node.js

Mimosa is written in node.js and you'll need to have it installed. You'll want to download and install the latest and greatest version. Anything over 8.0 will do. The node.js install comes with the Node Package Manager (npm).

2) mimosa

Use npm to install Mimosa.

$ npm install -g mimosa

The -g flag installs Mimosa globally so it can be run from the command line anywhere. You may need to use sudo to install this depending on the setup of your system.

Quick Start - Existing App easy to wire up

1) mimosa config

Execute mimosa config from the root of your application and Mimosa will place a mimosa-config.coffee file there. This file controls everything Mimosa does. For now its contents will be entirely commented out.

While Mimosa doesn't currently deliver its config as a JavaScript file, you are welcome to transform the mimosa-config.coffee to JavaScript. Mimosa is fine with that. Just transform the CoffeeScript comments to JavaScript comments. Should you start uncommenting config, you'll have a bit more to do to make the file valid JavaScript.

2) Tweak the config

From here its a matter of either changing your codebase around a little to match Mimosa's defaults, or changing Mimosa's config to work with your setup. The mimosa-config is self-documenting. So all you need to understand each config setting is inside the file. But you can also breeze through the configuration options on the site to understand all that you have control over.

The most important settings in the config, to get you started, are inside the top watch setting. Make sure that sourceDir, compiledDir, and javascriptDir all point to the right place in your codebase.

3) Fire it up

Execute mimosa watch and Mimosa will start keeping track of your assets and do its thing when you change them. Mimosa will tell you if you've done anything wrong, or if it can't find the important things it needs to find.

4) Next steps?

  • Get familiar with Mimosa's commands
  • Writing plain CSS or JS? Not using templating? Browse the list of compilers to see what your options are there.
  • Check out how Mimosa can help serve your assets or start your server.
  • Learn how Mimosa can optimize your code.
  • Having trouble? Hit the Google group to find a solution.

Quick Start - New App you'll be coding soon

1) Create a new application

Navigate to a directory within which you want to place your application and use mimosa new to create an app.

$ mimosa new nameOfApplicationHere

Follow the prompts and choose the meta-language, server, and server view technologies you'd like to use. If you aren't sure which ones to choose, that's perfect. Mimosa is a great tool to learn them. For now you can go with the ones marked with an asterisk as they are Mimosa favorites.

2) Start the server

Change into the directory that was created and execute:

$ mimosa watch --server

In your browser navigate to http://localhost:3000 to see the sample app. Mimosa will watch the assets directory of your new project and compile the changes you make over to the public directory. The web app serves up the assets in the public directory.

3) Start Coding

You are ready to go! So that you have a few examples to get you started, Mimosa has delivered you a bare bones web app with some pieces of working code using your chosen technologies.

Head over to the Commands tab to see how to interact with Mimosa, and figure out all it can do for you.

If you want to play around with something meatier than the codebase that mimosa new delivers, try out a demo app.

Demo Applications

Before starting your own application, play around with someone else's! Download the source from GitHub and use Mimosa's command line interface to play with the code and get comfy with both how Mimosa works and with the technologies inside the demo apps.

As more demo apps are built, they'll get added here. If you put a little something together using Mimosa let us know so we can add it to the list! Either open an issue or shoot a message to @mimosajs.
