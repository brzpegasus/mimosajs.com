Getting Started

Installation, and first steps. You are just minutes away from coding.

Installation it's a piece of cake

1) node.js

Mimosa is written in node.js and you'll need to have it installed. You'll want to download and install the latest and greatest version. Anything over 8.0 will do. The node.js install comes with the Node Package Manager (npm).

2) mimosa

You use npm to install node packages.

$ npm install -g mimosa

The -g flag installs Mimosa globally so it can be run from the command line anywhere. You may need to use sudo to install this depending on the setup of your system.

3) All done!

Quick Start you'll be coding soon

1) Create a new application

The easiest way to get started with Mimosa is to use it to create a new application skeleton. Mimosa will create a basic web application configured to match your desired meta-languages.

First navigate to a directory within which you want to place your application. Create the default app:

$ mimosa new nameOfApplicationHere

Follow the prompts and choose the meta-languages and server you'd like to use. If you aren't sure which ones to choose, that's perfect. Mimosa is a great tool to learn these technologies. For now you can go with the ones marked with an asterisk.

2) Start the server

Change into the directory that was created and execute:

$ mimosa watch --server

In your browser navigate to http://localhost:3000 to see the sample app.

Mimosa will watch the assets directory of your new project and compile the changes you make over to the public directory. The web app serves up the assets in the public directory.

3) Start Coding

You are ready to go! So that you have a few examples to get you started, Mimosa has delivered you a bare bones web app with some pieces of working code using your chosen pre-compilers.

If you want to play around with something meatier than the codebase that mimosa new delivers, try out a demo app.

Demo Applications

Before starting your own application, play around with someone else's! Download the source from github and use Mimosa's command line interface to play with the code and get comfy with both how Mimosa works and with the technologies inside the demo apps.

As more demo apps are built, they'll get added here. If you put a little something together using Mimosa let us know so we can add it here! Either open an issue or shoot a tweet with a link to @mimosajs.
