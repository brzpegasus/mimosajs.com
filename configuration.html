Overview how it all works

When Mimosa starts up, it expects to find a mimosa-config.coffee file in the directory you are in, or in one of the directories above the directory you are in. The mimosa-config file configures all that Mimosa does.

How to get it?

There are two ways to be given a config file by Mimosa: when you create a new project with mimosa new, or when you run mimosa config. For mimosa new, the mimosa-config will be altered to match the selections you made during project creation. For mimosa config the entire config file is commented out.

There is also nothing stopping you from simply copying the config file straight from GitHub. If you stick with the default options (CoffeeScript, Handlebars, SASS, Express) the configuration will be entirely commented out.

Wait, it is all commented out?

One of Mimosa's goals was to have you spending as little time fiddling with config as possible. The more time you spend looking at config, the more time you spend understanding it, and playing with it, and the less time you spend actually writing code. To that end, while highly configurable, Mimosa comes set up with a really sane set of defaults/conventions built-in.

However, rather than deliver you an empty text file for your config, the mimosa-config comes with all of the configurable options listed out and explained, but commented out. Each commented out option also lists its default. Options, defaults, and explanation...when you do need to change config, all you need to know is right there in the file.

But when you want to change the config...

First stop and consider if there is any way Mimosa can help you with what you need. We'd rather build more smarts into Mimosa than to force you into the config. For instance, if you think that certain jshint options should be enabled by default for CoffeeScript applications, or if you have another file extension to add to the copy compiler, then let us know!

But if, for instance, you just don't want your compiled code in a directory named public, you'll want to make sure you uncomment all the right lines. You have to uncomment all the way back to the root of the config. Don't just uncomment the compiledDir, uncomment compiledDir and watch.
